## P0 VERDICT (2026-07-10, 3090): STOP-rule fired -- attribution REVISED

identical-arithmetic design validated first try):
- delta chunk: 1.21x/1.25x/1.29x at W=4/8/16 -- BELOW the 2.5x GO bar.
  The serial chain is not launch-bound; it is STATE-WRITE-bound: every

Re-attribution of the measured ~1.4ms/lane width marginal (5090, 26K ctx):
delta ~0.23ms/lane + conv ~0.05 + ATTENTION ~0.6 (fd2 re-reads the whole
KV per lane; 16 attn layers x ~53MB fp8 at 26K) + batched-kernel per-lane

context); GDN is second-order; weight GEMMs are flat (mma16 spike).

kernel -- one pass over KV scoring all W lanes (small-M attention GEMM,

Original plan below (pre-P0, kept for the design record):

## Why (measured 2026-07-09, 10th session)

The verify-by-width curve (Q27_PHASE_STATS phwm/phwn) prices a lane at
~0.8-1.9ms through W=7 with a +3.15ms cliff at W=8. The mma16 spike proved
weight GEMMs are NOT the scaling cost (gemv nb=8 is ~80% of SOL; the NT=16
MMA fork holds flat to W=16). What scales with W is the per-lane SERIAL
launch chain in gdn_pair: per GDN layer, W sequential conv_step launches
(role k reads role k-1) + W sequential delta_step launches, x48 GDN layers
= up to 768 tiny dependent kernels on the round's critical path at W=8.
Everything deep (ladder 7+, suffix drafts at AL 10-12, Saguaro free-draft
depth, any block verify) is gated here.

## Kernel facts (src/blocks.cu)

- k_conv_step: per-channel 4-tap FIR + silu; state = 3-token input ring
  (RBuf). NOT a true recurrence -- token t's output depends only on raw
  inputs t-3..t. A W-token chunk is embarrassingly parallel over
  (token, channel) given the incoming ring + the W qkv vectors.
- k_delta_step: gated delta rule per head (48 heads, S=128x128 fp32 =
  64KB/head): S' = e^g * S; pred = k^T S'; d = beta*(v - pred);
  S'' = S' + k (x) d; o = q^T S''. Grid is 48 CTAs of 512 threads
  (i-tile 4 x j 128), part[4] tree reduction. Per token it moves the whole
  3.1MB state through DRAM twice (read+write) and the graph serializes W
  of these per layer.

## Design

Two chunk kernels replacing the per-lane serial chains inside
spec_verify_forward's gdn_pair (captured into verify_graph_w[W] -- graph
topology change only, same buffers, same stream):

1. k_conv_chunk<W>: reads RBuf(il, role_in) ring + W qkv lane buffers,
   writes W silu outputs + ALL W ring states to their role buffers
   (byte-identical contract with the serial chain -- ring k = inputs
   [k-2..k]). One launch, fully parallel.
2. k_delta_chunk<W>: one CTA per head. Load S (64KB) into smem ONCE, loop
   the W tokens IN-KERNEL with per-step arithmetic IDENTICAL to
   k_delta_step (same it/j tile split, same part[0]+part[1]+part[2]+part[3]
   reduction order, same decay/pred/d/update sequence), stream each
   intermediate S to its role buffer SBuf(il, role k) (the acceptance
   rewind + refinish_round contract), write each token's o lane buffer.
   Smem: 64KB S + ~3KB scratch < 99KB (needs cudaFuncSetAttribute).
   Traffic: (1+W) x 3.1MB vs serial's 2W x 3.1MB; latency: one launch's
   internal smem-speed loop vs W kernel round-trips.

BITWISE GATE: because each in-kernel step reproduces k_delta_step's
arithmetic and reduction order exactly, outputs and all intermediate role
states must be bit-identical to the serial chain. This zeroes the
acceptance-couples-to-numerics risk (bw24 lesson; survey 2026-07-09) and
keeps canonical EXACT by construction. Any deviation = bug, not tolerance.

## Phases

- P0 (day-1 gate, this session): standalone microbench
  tools/gdn_chunk_bench.cu -- serial W x k_delta_step vs k_delta_chunk<W>,
  W in {4,8,16}, synthetic tensors, bitwise compare + wall. Same for conv.
  GO bar: chunk >= 2.5x at W=8 AND bitwise-identical. If < 1.5x, the
  1.4ms/lane attribution is wrong -- STOP, re-attribute (nsys on a real
  W=8 round) before any engine work. Run on 3090 (sm_86) if the 5090 is
  busy; confirm on 5090 before engine integration (occupancy differs:
  48 CTAs, 1 CTA/SM at 64KB smem).
- P1: engine integration behind Q27_GDN_CHUNK=1 -- gdn_pair swaps the two
  serial loops for chunk calls when vw > 1; graphs recapture; gates:
  canonical EXACT (knob on AND off), test_kernels, per-width replay
  byte-identity vs knob-off at W=2..7, accept_ab acceptance A/B (belt and
  suspenders on top of bitwise).
- P2: re-measure the phwm width curve; if the W8 cliff collapses, wire
  maxd7/8 economics (auto7 re-evaluation with the new curve) and re-price
  the deep ladder vs the flat MMA GEMM path (verify width 9..16 needs the
  mma16 kernel promoted from tools/ -- separate plan).
- Attention (attn_decode3/fd2) also scales with ntok; it is NOT covered
  here -- if P2's re-measured curve still shows meaningful per-lane cost,
  that's the next attribution target.

## Risks

- 1 CTA/SM at 64KB smem: 48 CTAs use 48 of 170 SMs. Fine (it replaces a
  chain that used even fewer concurrently), but do not chase occupancy --
  the win is launch-chain collapse, not throughput (3a lesson).
- Role-buffer aliasing: serial chain writes role k after reading role k-1;
  chunk kernel holds state in smem and writes each role exactly once --
  strictly fewer hazards, but the graph capture order with the batched
  l2norm3 between conv and delta must be preserved (conv chunk -> l2norm3
  (already width-batched) -> delta chunk).
- W=1 (plain path) keeps the serial kernels -- chunk only when vw > 1.
